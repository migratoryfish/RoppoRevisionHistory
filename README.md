# 民法 改正履歴ビューア

日本の民法の条文改正履歴を、GitHubの差分表示風UIと時系列ビジュアライゼーションで閲覧できる静的Webアプリ。

データソースは [e-Gov 法令API v2](https://laws.e-gov.go.jp/api/2/swagger-ui)。
収録範囲はe-Govが保持する2016年10月以降（未施行の将来改正を含む）。時間軸は**施行日**ベース。

## 使い方

```bash
npm install
npm run fetch-data   # e-Gov APIから全バージョンをダウンロード（data/raw/ にキャッシュ）
npm run build-data   # public/data/minpo.json を生成
npm run dev          # 開発サーバー
npm run build        # dist/ に静的サイトを出力（任意の静的ホスティングに置くだけで動く）
```

## 機能

1. **GitHub風差分表示** — 項・号を「行」として行単位diff、変更行内は文字レベルのインライン差分（word diff相当）を赤/緑ハイライト
2. **改正マップ（ヒートマップ）** — 章 × 施行日のグリッドで、いつ・どこが・どれだけ変わったかを俯瞰。セルクリックで変更条一覧へ
3. **条文タイムライン** — 時間比例軸に改正イベントをプロット。未施行の将来版は中抜きドット、「現在」線付き
4. **全文＋由来（blame）** — 各項・号がどの改正で導入されたかを左帯で表示（git blame相当）。帯クリックでその改正の差分へ

## データモデル

- **スナップショット方式**: 施行日ごとの全文を保持し、diffは表示時にクライアントで計算（`diff` パッケージ）
- 同一施行日に複数の改正法が施行される場合は1スナップショットに集約
- 条ごとに「内容が変わったスナップショットのみ」本文を保存 → 民法全historyで約1MB

## 条番号の変動について

民法は削除条文が「第○条　削除」として残るため大半の条番号は安定しているが、
再編で条が消滅・新設されるケースがある（例: 2028-12-23施行予定の成年後見制度見直しで第13〜19条が消滅）。

現状の実装: 消滅を「消滅イベント」として明示し、**同じ施行日に新設された条を移動先候補として提示**する。

発展案（未実装）: 本文の編集距離による類似度で移動先を自動推定し、`pipeline/` に手動対応表（JSON）を
置いて上書きできるハイブリッド方式。

## 他の法令への拡張

`pipeline/fetch.mjs` の `LAW_ID` を変えれば他の法令にもそのまま使える（法令IDはe-Gov法令検索のURLから取得）。
