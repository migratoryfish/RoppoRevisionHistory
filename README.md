# 法令改正履歴ビューア

日本の法令の条文改正履歴を、GitHubの差分表示風UIと時系列ビジュアライゼーションで閲覧できる静的Webアプリ。

公開: https://migratoryfish.github.io/RoppoRevisionHistory/

データソースは [e-Gov 法令API v2](https://laws.e-gov.go.jp/api/2/swagger-ui)。
収録範囲はe-Govが保持する2016年頃以降（未施行の将来改正を含む）。時間軸は**施行日**ベース。

## 収録法令（43件）

六法＋主要法令＋司法書士業務法令。対象リストは `pipeline/laws.mjs` で管理。

- 憲法 / 民事（民法・商法・会社法・民訴法・破産法など17件）/ 刑事3件 / 行政5件 / 労働3件 / 知財4件 / 経済法2件
- 司法書士業務: 商業登記法・商業登記規則・不動産登記規則・会社法施行規則・会社計算規則・司法書士法・司法書士法施行規則・登録免許税法

**除外（データサイズ超過のため）**: 租税特別措置法（1版14.7MB）・所得税法（15.7MB）・法人税法・消費税法・地方自治法（196版）。
**登録免許税法は現行版のみ収録**（161版の履歴は取得しない。`latestOnly: true`）。

e-Gov非収録の文書は `data/manual/` に手動テキストを置く方式で収録する（書式は `data/manual/README.md`）。
現在は**民事保全規則**（裁判所公表PDFから変換・現行版のみ）を収録済み。民訴規則・民執規則・
不動産登記事務取扱手続準則はメタ情報のみ用意済みで、本文テキストを置けば表示される。

## 使い方

```bash
npm install
npm run fetch-data   # e-Gov APIから全法令・全バージョンをダウンロード（data/raw/<lawId>/ にキャッシュ、差分のみ再取得）
npm run build-data   # public/data/laws/*.json と public/data/index.json を生成
npm run dev          # 開発サーバー
npm run build        # dist/ に静的サイトを出力（任意の静的ホスティングに置くだけで動く）
```

## 機能

1. **GitHub風差分表示** — 項・号・表の行を「行」として行単位diff、変更行内は文字レベルのインライン差分を赤/緑ハイライト。**統合表示と新旧対照表（左=旧・右=新）の切替**可能
2. **改正マップ（ヒートマップ）** — 章 × 施行日のグリッドで、いつ・どこが・どれだけ変わったかを俯瞰。セルクリックで変更条一覧へ
3. **条文タイムライン** — 時間比例軸に改正イベントをプロット。未施行の将来版は中抜きドット、「現在」線付き
4. **全文＋由来（blame）** — 各項・号がどの改正で導入されたかを左帯で表示（git blame相当）。帯クリックでその改正の差分へ
5. **表・別表・別記** — 表（TableStruct）は行単位で取り込み差分対象。全文表示では実HTMLテーブルで描画。別表（AppdxTable）・別記（AppdxNote）・様式（AppdxStyle）・書式（AppdxFormat）は「別表・様式」グループとして収録
6. **様式PDF** — 条文・様式が参照する添付ファイル（登記の様式PDF等）を取り込み、折りたたみプレビューで表示。新旧対照モードでは旧様式・新様式が左右に並置される

## 自動更新（GitHub Actions）

`.github/workflows/update.yml` が毎週月曜 06:00 JST に実行され、
e-Govから増分取得 → データセット再生成 → 変更があれば main へコミット → gh-pages へデプロイまで自動化。
手動実行は Actions タブの「データ更新とデプロイ」→ Run workflow。
生データ（data/raw、約500MB）は actions/cache に保持され、毎回の再取得はしない。

## データモデル

- **スナップショット方式**: 施行日ごとの全文を保持し、diffは表示時にクライアントで計算（`diff` パッケージ）
- 同一施行日に複数の改正法が施行される場合は1スナップショットに集約
- 条ごとに「内容が変わったスナップショットのみ」本文を保存
- 法令ごとに1ファイル（`public/data/laws/<lawId>.json`）を遅延ロード

## 条番号の変動について

削除条文は「第○条　削除」として残るため大半の条番号は安定しているが、再編で条が消滅・新設されるケースがある。
現状の実装では消滅を「消滅イベント」として明示し、**同じ施行日に新設された条を移動先候補として提示**する。
発展案（未実装）: 本文の編集距離による類似度で移動先を自動推定し、手動対応表（JSON）で上書きできるハイブリッド方式。

## 法令の追加方法

`pipeline/laws.mjs` に `{ id, category }` を1行追加して `npm run fetch-data && npm run build-data`。
法令IDはe-Gov法令検索のURLまたは法令APIで確認。追加前にリビジョン数と1版サイズの実測を推奨
（巨大法令は `latestOnly: true` で現行版のみにする）。
