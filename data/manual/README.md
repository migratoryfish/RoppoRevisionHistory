# 手動収録文書（e-Gov非収録の裁判所規則・通達準則）

e-Gov法令APIに収録されていない文書を、このディレクトリにテキストで置くと
`npm run build-data` がe-Gov由来の法令と同じデータセットに変換します。
**版のファイルを追加するだけで、差分・タイムライン・blameが自動的に有効になります。**

## ディレクトリ構成

```
data/manual/<slug>/
  meta.json          … タイトル・番号・出典
  2024-03-01.txt     … その日から適用される全文（複数置けば履歴になる）
  2026-04-01.txt     … 改正があったら新しい全文を追加
```

- `<slug>` は英数字とハイフン（URLの一部になります）
- 本文の `.txt` が1つもないディレクトリはスキップされます（本文未投入の状態）

## meta.json

```json
{
  "title": "民事保全規則",
  "num": "平成二年五月十六日最高裁判所規則第三号",
  "category": "規則・準則（手動収録）",
  "source": "出典URL",
  "sourceNote": "変換方法・注意点のメモ（画面に表示されます）"
}
```

## 本文テキストの書式

```
＃第一章　総則              ← 「＃」= 編・章（大見出し）
＃＃第一節　通則            ← 「＃＃」= 章・節（小見出し）
■第一条（申立ての方式）    ← 「■」= 条の開始。（見出し）は任意
次に掲げる申立ては、…      ← 項の本文（第1項は番号なし）
２　前項の呼出しが…        ← 第2項以降は番号から
　一　保全命令の申立て      ← 号は行頭の全角スペース1つ
　　イ　…                  ← イロハは全角スペース2つ
```

- 条の同一性は「第○条」の表記で判定するので、版をまたいで表記を揃えること
- PDFからの変換例は `pipeline/`（民事保全規則の変換に使ったスクリプトの手順はコミット履歴参照）
